# Parallel File Hasher

## Overview

This program splits a file into fixed-size blocks, computes a SHA-256 hash for each block, and optionally verifies integrity and measures performance both serially and in parallel using OpenMP. It supports two modes:

1. **create**: compute and write block hashes to an output “.hash” file  
2. **check**: read a previously generated hash file, recompute hashes serially and in parallel, verify integrity, and report timing, throughput, speed-up and per-thread efficiency

## Prerequisites

- A C compiler with OpenMP support (e.g. `gcc`, `clang`)  
- OpenSSL development libraries (`libssl-dev` or equivalent)  
- POSIX-compatible OS (for `pread`, `stat`, `clock_gettime`)  

## Building

Compile all sources into a single executable (here named `hasher`):

```bash
gcc -std=c11 -O2 -fopenmp -Wall \
    app.c cli.c file_hasher.c hash_block.c hashfile.c perf_stats.c timer.c \
    -lcrypto -o hasher
````

* `-std=c11` – use C11 standard
* `-O2` – optimize for speed
* `-fopenmp` – enable OpenMP
* `-lcrypto` – link OpenSSL

## Usage

```
Usage:
  hasher create -i INPUT    -o OUTPUT    -b BLOCK_SIZE [-t THREADS]
  hasher check  -i INPUT    -h HASHFILE  -b BLOCK_SIZE -t THREADS
```

* **create**

  * `-i INPUT`      : path to the input file to hash
  * `-o OUTPUT`     : path to write the hash file
  * `-b BLOCK_SIZE` : block size in bytes (must be > 0)
  * `-t THREADS`    : number of threads for parallel hashing (default: 1)

* **check**

  * `-i INPUT`      : path to the input file to verify
  * `-h HASHFILE`   : path to an existing hash file (generated by **create**)
  * `-b BLOCK_SIZE` : block size in bytes (must match the one used in **create**)
  * `-t THREADS`    : number of threads for parallel hashing

### Examples

1. **Create** a hash file using 4 MB blocks and 8 threads:

   ```bash
   ./hasher create -i video.mp4 -o video.hsh -b 4194304 -t 8
   ```

2. **Check** integrity and measure performance on the same file:

   ```bash
   ./hasher check -i video.mp4 -h video.hsh -b 4194304 -t 8
   ```

   Sample output:

   ```
   Serial integrity: OK
   Parallel integrity: OK
   Serial   : 3.142857 s (128.00 MB/s)
   Parallel : 0.785714 s (512.00 MB/s)
   Speed-up : 4.00×
   Efficiency: 0.50 per thread
   ```

## Source File Breakdown

* **app.c**

  * Implements `main()` and `run_app()`.
  * Parses CLI options, chooses **create** or **check** mode.
  * In **create**, initializes `FileHasher`, computes parallel hashes, writes to file.
  * In **check**, reads `HashFile`, computes serial & parallel hashes, compares, and reports performance.

* **cli.c / cli.h**

  * `parse_cli()`: parses `argv[]` into `ProgramOptions`.
  * `print_usage()`: prints correct syntax when arguments are missing or invalid.

* **file\_hasher.c / file\_hasher.h**

  * `fh_init()`: stats file size, allocates hash buffer, determines `hash_len` by hashing zero bytes.
  * `fh_compute_serial()`: reads each block with `pread()`, hashes sequentially.
  * `fh_compute_parallel()`: same but with OpenMP `#pragma omp for`, one block per iteration.
  * `fh_free()`: releases buffers.

* **hash\_block.c / hash\_block.h**

  * `compute_hash()`: wraps OpenSSL’s EVP API to compute SHA-256 (`EVP_sha256`).

* **hashfile.c / hashfile.h**

  * `hf_write()`: writes header line (`block_size num_blocks hash_len`) and each block’s hash as a hex string.
  * `hf_read()`: reads header, allocates buffer, parses each hex-encoded hash line.
  * `hf_free()`: frees the read buffer.

* **timer.c / timer.h**

  * `timer_start()`: records `CLOCK_MONOTONIC` timestamp.
  * `timer_diff_s()`: computes elapsed seconds between two timestamps.

* **perf\_stats.c / perf\_stats.h**

  * `compute_throughput_MBps()`: MB/s from bytes and seconds.
  * `compute_speedup()`: ratio of serial to parallel time.
  * `compute_efficiency()`: speed-up divided by number of threads.

## Parameters and Data Structures

```c
typedef enum { MODE_CREATE, MODE_CHECK } Mode;

typedef struct {
    Mode    mode;
    char   *input_file;
    char   *hash_file;    // for check
    char   *output_file;  // for create
    size_t  block_size;
    int     num_threads;
} ProgramOptions;

typedef struct {
    const char *file_path;
    size_t      block_size;
    size_t      num_blocks;
    size_t      hash_len;
    unsigned char *hashes;  // contiguous array: num_blocks × hash_len
} FileHasher;

typedef struct {
    size_t      block_size;
    size_t      num_blocks;
    size_t      hash_len;
    unsigned char *hashes;  // as read from file
} HashFile;
```

* **block\_size** – in bytes, must match between create/check
* **num\_blocks** – computed as `ceil(file_size / block_size)`
* **hash\_len** – always 32 for SHA-256 (determined at runtime)

## Output Interpretation

* **Serial integrity** / **Parallel integrity**: whether recomputed hashes match the recorded ones.
* **Serial** / **Parallel**: total hashing time in seconds and throughput in MB/s.
* **Speed-up**: `serial_time / parallel_time`.
* **Efficiency**: `speed-up / num_threads`, measure of parallel utilization.

## Notes

* Block hashing order is deterministic: blocks numbered from 0 to N−1.
* Uses `pread()` so parallel threads read different file regions without interfering.
* OpenMP threads each allocate their own input/output buffers.
* If you encounter “Error …”, check file paths, permissions, block size, and thread count.
